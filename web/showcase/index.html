What certAIn Project Intelligence does

The product combines LLM-assisted planning, schedule-risk simulation, and scenario recommendation in one stakeholder-facing workflow. The objective is not to replace PM tools; it is to improve confidence before commitment.

1

Generate

Convert a project brief into a structured task plan with a resilient mock fallback.

2

Validate

Apply schema and DAG checks so malformed or cyclic plans never reach the forecast engine.

3

Forecast

Run dynamic critical-path Monte Carlo simulations to estimate mean, P50, P80, and delay probability.

4

Diagnose

Rank delay drivers by critical-path frequency so mitigation effort goes to the real bottlenecks.

5

Compare

Evaluate baseline, aggressive deadline, and increased-capacity scenarios in one decision view.

6

Explain

Surface an executive summary and advisory ML signal without pretending the classifier replaces the simulation.

Data and modeling summary

  • 0 missing values, 0 duplicate rows, 0 duplicate task IDs.
  • Target class split: Low 50.8%, Medium 29.0%, High 20.2%.
  • Selected advisory model: HistGradientBoosting.
  • Selection rule: highest cross-validated macro F1, not raw accuracy.

Why the ML model is advisory only

  • The dataset is clean, but class separation is weak.
  • Raw accuracy overstates performance because the target is imbalanced.
  • Monte Carlo remains the primary forecasting engine.
  • ML adds task-level directional risk context, not commitment certainty.
